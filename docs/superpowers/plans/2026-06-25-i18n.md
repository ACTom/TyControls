# ty-controls Internationalization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make ty-controls' translatable surface (design-time IDE strings, theme diagnostics, the main demo) internationalized via `resourcestring` + GNU-gettext `.po`, shipping English (base) + Simplified Chinese (zh_CN).

**Architecture:** Central `*StrConsts` per package (LCL `lclstrconsts.pas` model). Runtime diagnostics → one new `tyControls.StrConsts` unit (interface `resourcestring`s) referenced by ThemeLint/Css.Parser/Css.Values/StyleModel; one `tycontrols.strconsts.pot`/`.po`. Design-time strings → `resourcestring`s in `tyControls.Design`; the IDE translates them from the dt package's `.po` at registration (`DoTranslatePackage`). Demo → English-base resourcestrings + `EnableI18NForLFM`-harvested `.lfm` captions + a runtime language switcher (`LCLTranslator.SetDefaultLang`).

**Tech Stack:** FPC `resourcestring`; LazUtils `Translations` (`TPOFile`/`TranslateUnitResourceStrings`); LCL `LCLTranslator` (`SetDefaultLang`); package/project i18n option (`EnableI18N` + `OutDir="languages"`); fpcunit headless tests.

**Spec:** `docs/superpowers/specs/2026-06-25-i18n-design.md`

**Baseline:** branch `feat/i18n` (off `feat/ide-new-items`). Tests currently 951 run / 0 failures / 11 known headless win32-1407 errors. Build commands: `lazbuild tycontrols.lpk`, `lazbuild tycontrols_dt.lpk`, `lazbuild examples/demo/demo.lpi`, `lazbuild tests/tytests.lpi`; run tests `./tests/tytests.exe -a --format=plain`.

**Conventions used below:**
- resourcestring id = `rs` + Area + ShortName. msgid = English/ASCII. Composed messages use `Format(rs…, [...])` so Chinese can reorder.
- `translate=false` entries (deep CSS-syntax errors, per spec decision 5) still BECOME resourcestrings but their `.po` `msgstr` stays equal to the English `msgid` (gettext shows English).
- All `.po` files UTF-8; never touch `SystemCharSetIsUTF8`. zh translations below are **drafts** — the user (native zh, maintains LCL zh_CN) reviews them when authoring the `.po`.

```pascal
procedure TDemoMainForm.InitLanguages;
const
  cLangDir = 'languages';           // ships next to the exe
begin
  LangCombo.Items.Clear;
  LangCombo.Items.Add('English');   // index 0 -> 'en'
  LangCombo.Items.Add('中文(简体)'); // index 1 -> 'zh_CN'
  // Default follows the OS locale; SetDefaultLang('') autodetects + loads the matching .po.
  SetDefaultLang('', cLangDir);
  if Pos('zh', LowerCase(GetEnvironmentVariable('LANG'))) > 0 then
    LangCombo.ItemIndex := 1
  else
    LangCombo.ItemIndex := 0;
end;

procedure TDemoMainForm.LangComboChange(Sender: TObject);
const
  cLangDir = 'languages';
begin
  case LangCombo.ItemIndex of
    1: SetDefaultLang('zh_CN', cLangDir);
  else
    SetDefaultLang('en', cLangDir);
  end;
  // SetDefaultLang retranslates resourcestrings + already-open form captions (TPOTranslator).
  // Re-assign the code-set greeting so it refreshes immediately on the live form:
  if LblHello.Caption <> '' then LblHello.Caption := rsDemoGreetingShown;
end;
```
